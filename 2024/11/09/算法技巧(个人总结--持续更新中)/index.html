快速幂

为了更好引入此概念,牛客上有一道经典的算法题,将它归类到位运算中(本质上就是这个).
题目如下:

链接:https://ac.nowcoder.com/acm/contest/996/A
来源:牛客网

题目描述
求a的b次方对p取模的值,其中0≤a,b,p≤10^9,p>0

输入描述:
三个用空格隔开的整数a,b和p.
输出描述:
一个整数表示a^b(mod)p的值.

看起来简单吧?只需要for循环再取mod就可收工了.但看看范围,很明显直接爆了或者超时.
你就算开long long 都没用!

分析:

本质上问题的解决还是在于a^b,这太大了,以至于分开求模再次方也无济于事.
这时候,有没有一种少一些循环且高效的算法呢?
不妨想想计算机是如何存储数据的.
不妨设想b为2的n次方.

就令b=64.演示如下:

a^1*a^1=a^2
a^2*a^2=a^4
a^4*a^4=a^8
a^8*a^8=a^16
a^16*a^16=a^32
a^32*a^32=a^64

本质是倍增原理,将a的数量翻倍.
那么不为2的情况呢?

来个b=105.那么…

a^105=a*a^8*a^32*a^64

(105)10=(1101001)2
(1)10  =(0000001)2
(8)10  =(0001000)2
(32)10 =(0100000)2
(64)10 =(1000000)2
//综合起来,看下伪代码:

function binexp(a,b)
    r=1
    while b!=0 //从低到高不断遍历每一位
       if n mod 2==1 //如果遍历是1 优化:n&1
          r = r*a //乘上各循环所对应的a的倍增数
       a =a*a //倍增a
       b=[b/2]//优化:b>>1
    return r

 时间复杂度;O(logn)

呼,接下来不就简单了吗?

#define _CRT_SECURE_NO_WARNINGS
#include <cmath>
#include <map>
#include <iostream>
#include<cstring>
#include <stdio.h>
#include <stdlib.h>
#include<ctype.h>
#include<algorithm>
using namespace std;

int main() 
{
long long a, b, p,ans;
cin >> a >> b >> p;
ans = 1 % p;
while (b)
{
    if ((b & 1) != 0)
    {
        ans=ans*a%p;
    }
    a = a * a % p;
    b = b >> 1;
    
}
cout << ans;


return 0;
}

快速乘

嘻嘻,同样类型,思路上是一样的但是细节上还是略有差别.
还是举例牛客,看题!

链接:https://ac.nowcoder.com/acm/contest/996/C
来源:牛客网

题目描述
求a乘b对p取模的值,其中1≤a,b,p≤10^18
输入描述:
第一行a,第二行b,第三行p.
输出描述:
一个整数,表示a×b(mod)p的值.

有了上一题的基础,理解起来想必容易很多.
但算法的细节还是不一样的.
这是将后面一个乘数转化为二进制形式计算.

举个例子吧!

12*11%5
12*11=12*(1011)2=12*2^3+12*2^1+12*2^0=96+24+12=132
用&与(>>,<<)得出
计算转换为:
((12%5)+(24%5)+(96%5))%5=(2+4+1)%5=2

现在就好理解了.

int ksc(long long a,long long b,long long p)
{
      long long result=0;
      while(b)//与上一题一样,从低到高不断遍历每一位
      {
            if((b&1)!=0)
            {
            result=(result+a)%p;//这样理解了为啥是加而不是乘
            }
      b=b>>1;
      }
   return result;
}
